# Overview-Builder Conversational Flow Wireframes

**Version:** 1.0
**Date:** 2026-02-24
**Status:** Draft

---

## Overview

This document defines the conversational user flow for the Overview-Builder add-in. The add-in guides users through creating custom dashboards using a 5-step process with contextual help and smart defaults.

---

## Design Principles

1. **Conversational** - Natural language guidance, not form-filling
2. **Progressive disclosure** - Show complexity only when needed
3. **Smart defaults** - Pre-select the most common choices
4. **Quick exits** - Allow users to save/cancel at any step
5. **Visual preview** - Show what you're building in real-time
6. **Permission-aware** - Only show options user has access to

## Mobile Responsive Behavior

### Small Screens (< 768px)

- Stack recipe cards vertically
- Single column layout forced
- Full-screen preview
- Bottom sheet for save dialog

- 2-column recipe grid
- Two-column layout recommended
- Side drawer for module selector

- 3-column recipe grid
- Grid layout recommended
- Split view: config on left, preview on right

**Keyboard Navigation:**
- All interactive elements focusable

- Descriptive error messages
